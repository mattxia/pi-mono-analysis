# Pi 项目分析与扩展指南

要分析和扩展 Pi 项目，可以按照**「学习理解 → 模块化扩展 → 生态贡献」**的路径逐步深入，结合项目高度可扩展的架构设计，以下是具体的入手方向：

---

## 一、项目分析学习路径（从易到难）
### 1. 上手体验（入门）
- 先直接使用 `pi-coding-agent` CLI，熟悉核心功能：交互模式、工具调用、会话管理、扩展安装等
- 跑通官方示例：extensions 示例、SDK 示例、web-ui 示例，理解扩展机制的使用方式
- 阅读各包的 `README.md` 和 `/docs` 目录下的文档，重点关注扩展开发相关内容

### 2. 代码结构理解（中级）
```
优先阅读顺序：
├── packages/coding-agent/src/ → 了解上层应用如何组装核心能力
│   ├── cli/ → 入口和命令处理
│   ├── core/ → 会话管理、工具实现、事件总线
│   └── modes/interactive/ → TUI 界面组装
├── packages/agent/src/ → 理解 Agent 核心运行原理
│   ├── agent-loop.ts → 核心执行循环
│   ├── agent.ts → 对外 API 层
│   └── types.ts → 核心实体定义
├── packages/ai/src/ → 理解 LLM 抽象层设计
│   ├── stream.ts → 流式响应处理
│   ├── providers/ → 各厂商的适配实现
│   └── types.ts → 统一 API 类型定义
└── packages/tui/src/ → 理解终端 UI 渲染原理
    ├── tui.ts → 差分渲染核心
    └── components/ → 基础组件实现
```

### 3. 深入核心机制（高级）
- 理解 `agentLoop` 的完整执行流程：消息转换 → LLM 调用 → 工具执行 → 结果反馈的完整闭环
- 分析 `pi-ai` 的多 Provider 适配方案：如何屏蔽不同厂商 API 差异，统一事件格式
- 研究 `pi-tui` 的差分渲染和同步输出技术，如何实现无闪烁终端交互
- 学习上下文压缩、会话分支、跨模型交接等高级特性的实现

---

## 二、主要扩展方向
### 🟢 入门级扩展（无需修改核心代码）
#### 1. Coding Agent 生态扩展
- **自定义 Extension**：这是最常见的扩展方式，几乎可以实现任何功能：
  - 新增自定义工具（如数据库操作、API 调用、第三方服务集成）
  - 新增自定义命令、快捷键、UI 组件、主题
  - 实现自定义工作流：plan 模式、子 agent、权限控制、审计日志等
  - 参考 `packages/coding-agent/examples/extensions/` 下的30+官方示例
- **自定义 Skill**：按照 Agent Skills 标准编写声明式技能包，让模型自动调用
- **自定义 Prompt Template**：封装常用提示词，实现一键调用
- **发布 Pi Package**：将扩展打包发布到 npm/git，供其他人安装使用

#### 2. 前端集成
- 基于 `pi-web-ui` 组件快速搭建自定义 AI 聊天界面
- 集成到现有产品中：知识库问答系统、内部研发助手、客户服务机器人等
- 自定义消息渲染器、工具渲染器，适配业务场景的展示需求

---

### 🟡 中等级扩展（少量修改核心代码）
#### 1. LLM Provider 扩展
按照项目内置的 [添加新 LLM 提供商规范](https://github.com/badlogic/pi-mono/blob/main/packages/ai/README.md#adding-a-new-llm-provider)，支持更多厂商：
- 新增国内厂商：通义千问、文心一言、豆包、DeepSeek 等
- 新增开源模型部署：Llama 3、Qwen 2、GLM 4 等的直接适配
- 新增特殊认证方式：企业内部 SSO、角色化权限控制等

#### 2. TUI 组件扩展
给 `pi-tui` 新增通用组件：
- 表格组件、树形组件、图表组件
- 更丰富的 Markdown 渲染能力
- 更多终端图片协议支持

#### 3. Agent 能力扩展
给 `pi-agent-core` 新增通用能力：
- 新增记忆系统：长期记忆、向量检索集成
- 新增工具执行策略：工具调用规划、重试机制、fallback 策略
- 新增上下文管理策略：更智能的压缩算法、相关历史召回
- 支持多 agent 协作模式

---

### 🔴 高级扩展（核心架构级修改）
#### 1. 新应用层产品
基于核心能力打造新的上层应用：
- 类似 `pi-mom` 的 IM 机器人：企业微信、Discord、Telegram 机器人
- IDE 插件：VS Code、JetBrains 系列插件，将编码能力直接集成到编辑器
- 桌面端应用：基于 Electron 打包的跨平台 AI 助手
- 服务端部署版本：多用户协作的团队级 AI 助手平台

#### 2. 部署能力扩展
给 `pi-pods` 新增更多部署能力：
- 支持更多云服务商：阿里云、腾讯云、华为云等国内 GPU 厂商
- 新增 K8s 部署模式，支持集群级管理和自动扩缩容
- 新增模型热加载、流量负载均衡、自动降级等高可用能力
- 支持微调任务管理、数据集管理等 MLOps 能力

#### 3. 跨语言生态
- 开发 Python/Go/Rust 等其他语言的 SDK，接入 Pi 生态
- 实现跨语言的工具调用协议，支持不同语言开发的工具扩展

---

## 三、开发最佳实践
1. **遵守项目规范**：严格遵循 `AGENTS.md` 中的开发规则，`npm run check` 必须全部通过再提交
2. **优先使用扩展机制**：90%以上的功能都可以通过 Extension 实现，无需修改核心代码，减少后续合并冲突
3. **测试覆盖**：新增功能必须补充对应的测试用例，尤其是 LLM 相关逻辑需要做适配性测试
4. **版本兼容**：所有扩展尽量保持向前兼容，不破坏现有 API
5. **参考已有实现**：新增 Provider/工具/组件时，先参考已有类似实现的代码风格和设计模式

如果是第一次贡献，可以先从 **新增 LLM Provider、编写 Extension 示例、修复小 Bug** 这类任务入手，熟悉开发流程后再做更复杂的扩展。
